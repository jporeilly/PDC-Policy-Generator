How this app fits the workflow — and why the Registry exists

In PDC, the same three facts about a column — which business term it maps to, what tags it carries, and how sensitive it is — get decided in more than one place. The Glossary Generator owns the reviewed vocabulary; at export it writes a Classification Registry: one row per governed concept carrying the term, the governed tags, the sensitivity, and the detection seeds its scan induced from profiled data (value-format regexes like ^CSCU-\d{6}$, profiled reference-value lists). This app reads that Registry and authors PDC's Data Identification methods from it — so the term a method binds and the tags it stamps can never quietly diverge from the glossary. Everything runs locally and deterministically: no LLM, no network — the evidence was already gathered at scan time.

Authorturn each detection seed into an import-ready method (this page — working) Reconcileverify each concept's minted PDC term_id against a live PDC and bind authoring to the ids (the Reconcile page — working) Deployimport methods over the public API (v3) and trigger the DATA_IDENTIFICATION jobs (roadmap) Drift-checkcompare deployed Assign-Tags + PDC's live tag facet against the Registry's governed vocabulary (roadmap)

3 Retire the imported set ?

PDC 11.0's method list offers only View and Edit — no Delete. This removes your authored methods (dictionaries and patterns) over the same /graphql endpoint the UI uses, scoped to a name prefix so it can only ever touch your set. Built-ins are refused even if one carries the prefix. Deterministic ids mean a re-import is an upsert, so a clean re-run rarely needs this — reach for it when a term left the glossary and its orphaned method would otherwise linger.

Under the hood — the GraphQL calls (live)

Data Identification is backed by a graphql-compose-mongoose Apollo endpoint at {pdc}/graphql, authenticated by the same bearer token as the REST API. Introspection is disabled in production, but the generated CRUD field names are confirmed against PDC 11.0.0.
